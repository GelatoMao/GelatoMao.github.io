## 编译型语言与解释型语言

学习 Java 之前，先理解一个基本概念：编译型语言和解释型语言的区别。

**编译型语言**（如 C、C++、Go、Rust）在运行之前，需要通过编译器将整个源代码一次性翻译成机器码。程序运行时直接执行的是机器码，不再需要源代码或编译器。因此编译型语言通常执行速度更快，但每次修改代码后都需要重新编译，开发迭代的周期相对较长。此外，编译产物通常与目标平台绑定，跨平台需要针对不同平台分别编译。

**解释型语言**（如 Python、JavaScript、Ruby）则不需要预先编译。解释器在运行时逐行（或逐块）读取源代码并即时执行。这使得开发调试更加灵活——改完代码立刻就能运行，但由于每次执行都需要解释器参与翻译，运行速度通常不如编译型语言。

Java 是一个典型的"两者兼具"的例子：它先通过 `javac` 编译成字节码（bytecode），再由 JVM 的解释器或 JIT（即时编译器）在运行时将字节码转为机器码执行。Python 也会先编译成 `.pyc` 字节码文件再由虚拟机执行，JavaScript 的现代引擎（如 V8）同样大量使用了 JIT 编译技术。

简单概括：编译型语言"先翻译，后执行"，追求运行效率；解释型语言"边翻译，边执行"，追求开发灵活性。但随着 JIT 等技术的发展，两者在性能上的差距正在缩小，很多现代语言已经是混合模式了。

## JDK 工具链

安装 JDK 后，可以在 `JAVA_HOME/bin` 目录下找到一系列核心工具。在 macOS 上查看 JDK 安装位置最简单的方式是运行：

```bash
/usr/libexec/java_home
```

以下是几个最重要的可执行文件：

### java —— JVM 启动入口

`java` 是 JVM 的启动入口。当你运行 `java MyApp` 时，操作系统启动的是 JVM 这个进程，JVM 再去加载 `MyApp.class` 字节码文件，把字节码解释（或通过 JIT 编译）成机器码来执行。它负责内存管理、垃圾回收、线程调度等所有运行时工作。

### javac —— Java 编译器

`javac` 负责把 `.java` 源码翻译成 JVM 能理解的 `.class` 字节码文件。比如执行 `javac Hello.java` 后会生成 `Hello.class`。这里的"编译"并不是编译成某个特定平台的机器码，而是编译成平台无关的中间格式（字节码），这也是 Java "一次编写，到处运行"的基础。

### jar —— 打包工具

`jar` 本质上类似于 `zip`，将大量 `.class` 文件和配置文件、资源文件打包成一个 `.jar` 文件，里面还包含一个 `META-INF/MANIFEST.MF` 清单文件用于指定主类等元信息。打包后只需分发一个 `.jar` 文件，运行时用 `java -jar app.jar` 就能启动。我们平时用的各种第三方库（比如 Spring、Guava）也都是以 `.jar` 的形式发布的。

### javadoc —— 文档生成工具

`javadoc` 会扫描源码中 `/** ... */` 格式的文档注释，自动生成 HTML 格式的 API 文档。Java 标准库的官方文档就是用这个工具生成的。注释中可以使用 `@param`、`@return`、`@throws` 等标签来描述方法的参数、返回值和异常。

### jdb —— Java 调试器

`jdb` 是命令行调试器，支持设置断点、单步执行、查看变量值、查看调用栈等。类似于 C/C++ 中的 GDB。实际开发中大多数人会用 IDE 内置的图形化调试器，但 `jdb` 在没有图形界面的服务器环境下仍然有用。

## 变量类型

Java 的变量类型分为两大类：**基本类型（primitive types）** 和 **引用类型（reference types）**。

### 基本类型

基本类型共 8 种，按用途分为四组：

**整数类型** —— `byte`（1 字节，-128 ~ 127）、`short`（2 字节，-32768 ~ 32767）、`int`（4 字节，约正负 21 亿，最常用）、`long`（8 字节，赋值需加 `L` 后缀，如 `long x = 100000000000L`）。

**浮点类型** —— `float`（4 字节，约 6-7 位有效数字，赋值需加 `f` 后缀）、`double`（8 字节，约 15-16 位有效数字，浮点数默认类型）。需要注意浮点数存在精度问题，`0.1 + 0.2` 不会精确等于 `0.3`，金融计算应使用 `BigDecimal`。

**字符类型** —— `char`（2 字节，单引号表示，采用 UTF-16 编码，可存中文如 `char c = '中'`）。

**布尔类型** —— `boolean`（只有 `true` 和 `false` 两个值）。

### 引用类型

引用类型包括类（class）、接口（interface）、数组和枚举（enum）等，变量存储的不是数据本身，而是对象在堆内存中的地址引用。常见的如 `String`、`Integer`、`List` 以及自定义类的实例。引用类型的默认值是 `null`。

### 两者的关键区别

基本类型直接存储值，存放在栈内存中，`==` 比较的是值本身；引用类型存储的是内存地址，`==` 比较的是地址是否相同（是否是同一个对象），比较内容需要用 `equals()` 方法。

Java 为每种基本类型都提供了对应的包装类（`Integer`、`Long`、`Double` 等），Java 5 之后引入了自动装箱和拆箱机制，基本类型和包装类之间可以自动转换。

## 数组与 Arrays 工具类

Java 中数组是一个对象，但它没有重写 `toString()` 方法，继承的是 `Object` 类的默认实现。如果直接打印数组：

```java
int[] ns = {28, 12, 89, 73, 65, 18, 96, 50, 8, 36};
System.out.println(ns); // 输出类似 [I@1b6d3586
```

`[I` 表示 int 数组，`@` 后面是哈希值，完全不可读。要打印数组内容，需要使用 `Arrays.toString()`：

```java
import java.util.Arrays;

int[] ns = {28, 12, 89, 73, 65, 18, 96, 50, 8, 36};
Arrays.sort(ns);
System.out.println(Arrays.toString(ns));
// 输出：[8, 12, 18, 28, 36, 50, 65, 73, 89, 96]
```

`Arrays.toString()` 会遍历数组中的每个元素，拼接成可读的字符串。这是 Java 数组的一个常见"坑"，不像 Python 的列表直接 `print` 就能看到内容。

## 可变参数（Varargs）

可变参数使用 `类型... 参数名` 语法定义，允许调用方法时传入任意个数（包括零个）的同类型参数，编译器会自动将其打包成数组：

```java
void sum(int... nums) {
    int total = 0;
    for (int n : nums) {
        total += n;
    }
    System.out.println(total);
}

// 以下调用方式都合法
sum(1, 2, 3);          // 编译器转为 sum(new int[]{1, 2, 3})
sum(1);
sum();
sum(new int[]{1, 2});  // 直接传数组也可以
```

在方法内部，可变参数本质就是一个数组，可以用 `.length` 获取长度、用下标访问、用 for-each 遍历。

使用可变参数需要注意几点：可变参数必须是方法的最后一个参数，比如 `void foo(String name, int... scores)` 合法，但反过来不行；一个方法最多只能有一个可变参数；传入 `null` 时 `args` 本身会是 `null`（不是空数组），直接遍历会抛 `NullPointerException`，而不传参数时则是长度为 0 的空数组。

常见的可变参数用法有 `String.format(String format, Object... args)`、`Arrays.asList(T... a)` 等，好处是让方法调用更简洁，不需要调用者手动创建数组。
