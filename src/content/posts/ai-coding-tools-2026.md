## 前言

AI Coding 已经从"尝鲜"变成了日常开发的基础设施。2026 年市面上的工具越来越多，各有侧重，这篇文章记录一下我目前的工具组合和使用心得。

## 我的工具栈

### CatPaw Desk

公司内部的 AI 编程助手，最大的优势是和内部系统深度集成。写代码、查文档、发消息、管理日程都能在一个地方搞定。对于日常工作来说效率非常高。

### Cursor

目前最顺手的 AI IDE。它的 Tab 补全非常智能，几乎能猜到你下一步要写什么。多文件编辑能力也很强，重构代码时特别方便。

### GitHub Copilot

老牌选手，胜在稳定。作为 VS Code 插件使用门槛最低，代码补全的质量一直在线。适合不想折腾、只需要基础补全能力的场景。

## 使用心得

### 什么时候用 AI 最有效

经过大半年的实践，我发现 AI Coding 在这些场景下收益最大：

- **写样板代码**：重复性高的代码让 AI 生成，比如 CRUD 接口、表单组件、类型定义
- **代码转换**：比如把 JavaScript 转 TypeScript、把 Options API 转 Composition API
- **写测试**：给一个函数让 AI 补全测试用例，覆盖各种边界情况
- **学习新框架**：让 AI 解释代码、生成示例，比看文档快得多

### 什么时候不该依赖 AI

- **核心业务逻辑**：AI 不了解你的业务上下文，生成的代码可能看起来对但逻辑是错的
- **性能敏感代码**：AI 生成的代码往往是"能用"但不是"最优"的
- **安全相关代码**：认证、加密、权限这些不能偷懒，必须自己把关

## 一个实际的例子

前几天我需要给项目加一个文件上传组件，用 AI 来辅助的完整流程大概是：

1. 先用自然语言描述需求："写一个支持拖拽上传、限制文件大小 10MB、只允许图片格式的 React 组件"
2. AI 生成初版代码，大概能覆盖 80% 的需求
3. 我来调整样式、补充错误处理、对接实际的上传接口
4. 让 AI 帮忙写单元测试

整个过程大概 20 分钟搞定，如果全部手写可能要一个多小时。

## 总结

AI Coding 不是替代程序员，而是放大程序员的能力。关键是找到适合自己的工具组合，知道什么时候用、什么时候不用。工具在快速迭代，保持关注但不必每个都追，选 2-3 个深度使用就够了。
